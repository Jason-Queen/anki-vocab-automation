# Study Session Rules

## Goal

这个 skill 的目标不是“代替 Anki 复习”，而是基于用户已有卡片和复习记录，做一段短而有针对性的陪练。

## Session Shape

推荐一轮只练 `5-10` 个词。

每个词优先使用这些题型之一：

- 看英文词，说中文义或简单英文义。
- 看带空的 context 例句，补目标词。
- 看 context 例句，说这个词在句子里的意思。
- 用户自己造句，agent 只做轻量纠偏。

不要把一轮会话做成考试系统。重点是帮助用户回忆和发现不稳点。

## Context Sentence Rule

- 优先使用卡片当前 `Example` 字段里的句子。
- 只有 `Example` 为空时，才退到 `GeneratedExample`。
- 没有可用例句时，再使用非 context 题型。

这样可以把练习绑在用户当初加卡时的语境上，减少“认得词但认不出当前用法”的问题。

## Prompt Variety

- 同一轮里要随机切换题型。
- 如果上一题已经是高负担输出题，下一题优先换成识别或回忆题。
- 如果某张卡有现成 context 例句，优先使用基于该例句的题型。

## Feedback Style

- 反馈要短。
- 先判断“意思对不对”，再判断“表达自不自然”。
- 对 gap-fill 或词形题，允许“词义方向对，但词形不对”的部分正确。
- 如果用户答案不够好，优先给一个更自然、更简单的版本。
- 不要把边缘错误说成完全错误；也不要把明显错义说成“差不多”。

## Partial-Correct Handling

- 如果用户回忆出了 lemma，但例句实际需要的是其他 surface form，先肯定部分正确，再指出形式差异。
- 典型例子是：
  - 单数 vs 复数
  - 原形 vs 过去式
  - 原形 vs `-ing`
- 在这类题里，优先使用 `scripts/study_turn_assist.py` 做确定性的 form check。

## Hint Mode

- 用户说“不会”“忘了”“不确定”时，不要立刻公布答案。
- 默认提示顺序：
  - 第 1 层：词性
  - 第 2 层：首字母 / 长度，或 context clue
  - 第 3 层：必要的词形提示
  - 最后：再给完整答案
- 如果用户已经给出接近答案，下一步优先给 form hint，而不是退回到更早的通用提示。

## Personalization Rules

- 只有在确实读到了 `cardsInfo` / `getReviewsOfCards` 的结果时，才可以说“根据你的复习记录”。
- 如果只有 deck 内现状，没有 review 历史，就说“根据现有卡片内容和基础状态”。
- 如果连可用卡片都不足，就直接说明，而不是编造候选词。

## Safety Boundary

- 默认只读，不替用户提交 Anki 评分。
- 默认不修改 note、tag、model、deck。
- 默认不声称用户“已经掌握”某词；最多说“近期表现更稳定”。
- 如果一个词有明显多义，练习时优先绑定卡片现有定义和例句，不要擅自扩展到其他词义。
